# タグサジェストシステム 設計ドキュメント

## 概要
メモのタイトル・本文からキーワードを抽出し、最適なタグを自動提案する。AIやサーバー不要、完全ローカル完結。

## アーキテクチャ

### 関連ファイル一覧
| ファイル | 役割 |
|---|---|
| `Services/TagSuggestEngine.swift` | サジェストエンジン本体（スコアリング・単語抽出・学習） |

| `Models/TagFrequency.swift` | 単語×タグの出現頻度（SwiftDataモデル） |

| `Views/TabbedMemoListView.swift` | カラーパレット定義（72色）＋色名 |

## エンジン処理フロー（`TagSuggestEngine.suggest()`）

### 入力
- `title`: メモタイトル
- `body`: メモ本文
- `tags`: 全タグ（`@Query`で取得済み）
- `context`: SwiftData ModelContext
- `limit`: 各セクションの最大件数（デフォルト3）

### Step 1: 単語抽出（`extractWords()`）
1. タイトル全文 + 本文先頭200文字を結合
2. `CFStringTokenizer`（`kCFStringTokenizerUnitWord`）で形態素解析
3. 各トークンについて:
   - ラテン転写（ローマ字読み）を取得 → `lowercased()`で追加
   - 原文トークン（2文字以上）を `lowercased()` で追加
4. 重複排除して返す

**⚠️ 重要な実装注意点:**
- `CFStringTokenizer` はUTF-16単位で動作する
- 範囲指定には `CFStringGetLength()` を使う（`String.count` は不可 — 絵文字でズレる）
- 部分文字列の取得は `String.UTF16View` のインデックスを使う
- これを間違えると「🍺 病院」→「院に」のような誤抽出が起きる（過去のバグ）

### Step 2: 辞書マッチ（6層スコアリング）

#### ① 事前辞書マッチ（完全一致 + 部分一致）
- **辞書構造**: `{ "キーワード": ["カテゴリ名1", "カテゴリ名2"] }`
- 辞書キーは起動時に `lowercased()` で正規化済み
- **完全一致**（スコア +1.0）: 抽出単語が辞書キーに一致 → カテゴリ名で既存タグを検索
  - タグ名 == カテゴリ名 or タグ名.contains(カテゴリ) or カテゴリ.contains(タグ名) → スコア加算
  - マッチしないカテゴリ → `newTagCandidates` に蓄積（新規タグ提案用）
- **部分一致**（スコア +0.5）: 入力単語と辞書キーが互いに含む
  - **ノイズ防止条件**: 両方3文字以上 ＆ 短い方が長い方の50%以上の長さ

#### ② ユーザー学習（TagFrequency）
- `TagFrequency` テーブルから、抽出単語に紐づくタグIDを検索
- スコア = `log2(count + 1) * 0.5`（対数スケールで上限抑制）

#### ③ 時間帯加重
- `TagFrequency.hour` と現在時刻を比較（±2時間以内でブースト）
- 同じ曜日ならさらにブースト

#### ④ 連続入力パターン
- `recentTagIDs`（直近10件のタグID）に基づくブースト
- 直近ほど高スコア: `2.0 / (i + 1)`

#### ⑤ 共起タグ加重
- `TagCooccurrence` テーブルから、現在スコアのあるタグと共起したタグを検索
- スコア = `log2(count + 1) * 0.3`

#### ⑥ 否定学習（スコア減算）
- `TagSuggestDismissal` テーブルから、却下された単語×タグの組み合わせを検索
- スコア = `-count * 0.5`

### Step 3: 結果構築（3セクション）

スコアリング後、辞書スコアと学習スコアを分離して3種類の候補を構築:

#### セクション1: おすすめタグ（`dictMatch`）
- 辞書マッチでスコアが付いた既存タグのみ
- 親タグ単位でソート → 各親に紐づく子タグも全てリスト化
- 親+子セット + 親単体を候補に

#### セクション2: 新規タグ提案（`newTag`）
- 辞書にはあるが既存タグに無いカテゴリ名
- `newTagCandidates` のスコア順で上位N件
- UIでは緑の＋アイコン、「タグを作成」ラベル付き

#### セクション3: 履歴から（`history`）
- 辞書マッチが無いが学習スコアだけあるタグ
- 時間帯・曜日・連続入力・共起のみでスコアが付いたもの

## 辞書（TagSuggestDictionary.json）

### 構造
```json
{
  "キーワード": ["カテゴリ名1", "カテゴリ名2"],
  "筋トレ": ["運動", "健康"],
  "荷物": ["旅行", "買い物", "タスク"]
}
```

### カテゴリの種類
- **既存タグ名に一致するカテゴリ**（27種）: 仕事, アイデア, 買い物, 趣味, 健康, 会議, タスク, 経費, 企画, 営業, 開発, 人事, 総務, 広報, 法務, 経理, 品質, 教育, 海外, 保守, ギター, ランニング, 映画, アプリ, ビジネス, 食事, 運動
- **新規タグ提案用カテゴリ**（13種）: 旅行, 出張, 料理, ペット, 読書, 美容, 家事, 子育て, 車, 引越し, 資格, 投資, イベント

### 設計方針
- 1つのキーワードが複数カテゴリに紐づいてOK（例: 「予約」→趣味, 健康, タスク）
- 既存タグ名カテゴリと新規タグ提案カテゴリが同居可能
- 起動時に1回JSONパース（2656語で約120KB、数十ms以内）
- キーワードは動詞形・口語・略語・英語・ブランド名を含む

## 学習データモデル（SwiftData）

### TagFrequency
メモ確定時、`learn()` で自動記録。
```
word: String       — 抽出された単語（lowercased）
tagID: UUID        — 紐づくタグのID
count: Int         — 出現回数（確定のたびに+1）
hour: Int          — 最後に使った時間帯（0-23）
weekday: Int       — 最後に使った曜日（1=日〜7=土）
lastUsedAt: Date
```

### TagCooccurrence
同一メモに2つ以上のタグが付いた場合に記録。
```
tagID1: UUID       — ペアの片方（UUID文字列が小さい方）
tagID2: UUID       — ペアのもう片方
count: Int         — 共起回数
lastUsedAt: Date
```

### TagSuggestDismissal
サジェストの×ボタンタップ時に `dismiss()` で記録。
```
word: String       — その時入力中だった単語
tagID: UUID        — 却下されたタグのID
count: Int         — 却下回数
lastDismissedAt: Date
```

## UI（MainView内）

### 表示トリガー
- `titleText` or `inputText` の `onChange` → `triggerSuggest()`
- 1秒デバウンス（Task.sleep）
- ガード条件: タグサジェスト設定ON ＆ タグ未選択 ＆ 閉じてない

### 表示条件（`shouldShowSuggestions`）
- サジェスト設定ON ＆ タグ未選択 ＆ 未dismiss ＆ 候補あり

### 3セクション表示
```
┌─────────────────────────────┐
│                          ✕  │  ← 閉じるボタン
│ 🏷 おすすめタグ               │
│ ● 健康 > 運動               │  ← 既存タグ（色付き丸）
│ ● 健康                      │
│                             │
│ ＋ 新規タグ提案               │
│ ＋ 旅行  タグを作成           │  ← 緑背景＋＋アイコン
│                             │
│ 🕐 履歴から                  │
│ ● 仕事 > 会議               │
└─────────────────────────────┘
```

### タップ時の処理（`applySuggestion()`）

#### 既存タグの場合（dictMatch / history）
1. `viewModel.selectedTagID = suggestion.parentID`
2. 子タグがあれば `viewModel.selectedChildTagID = childID`
3. `viewModel.onTagChanged(tags:)` でルーレット・タグバッジに反映

#### 新規タグの場合（newTag）
1. `Tag(name:, colorIndex:)` で新規タグ作成
2. **色の自動割り当て**（`pickDistinctColor()`）:
   - 並び順で最後尾2つの親タグの色相を取得
   - 全候補色（1〜72）について、最後尾2色との色相距離（環状）の最小値を計算
   - 色相距離が最大 ＆ 既存タグで未使用のものを選ぶ
   - 全色使われてる場合は色相距離のみで選択
3. `modelContext.insert()` → `save()`
4. 作成したタグを選択状態にする

## カラーパレット（72色）

### 構成
- 0: タグなし（ノーカラー）
- 1〜7: 基本色（アクア, みかん, ピスタチオ, すみれ, レモン, ストロベリー, コバルト）
- 8〜14: パステル系（シャボン, ピーチ, ミント, ラベンダー, バニラ, サーモン, あじさい）
- 15〜21: 深め（ティール, テラコッタ, フォレスト, プラム, マスタード, ガーネット, インディゴ）
- 22〜28: アクセント（ターコイズ, コーラル, ライム, アメジスト, ゴールド, ローズ, ブルーベリー）
- 29〜35: ナチュラル系（サンド, セージ, モカ, アイボリー, オリーブ, キャメル, シルバーレイク）
- 36〜42: ビビッド系（チェリー, ソーダ, メロン, マンゴー, グレープ, トマト, エメラルド）
- 43〜49: くすみ系（モーヴ, ユーカリ, さくら, フォグ, カーキ, カメオ, しずく）
- 50〜56: レトロ系（シナモン, ヒスイ, ココア, ウェッジウッド, ハニー, ボルドー, ナイル）
- 57〜63: ポップ系（フラミンゴ, アクアマリン, キウイ, サンフラワー, オーキッド, パプリカ, ラムネ）
- 64〜70: スモーキー系（トワイライト, モス, クレイ, ミスト, サンドストーン, カシス, アイス）
- 71〜72: 特別色（シャンパン, フォレストミスト）

### 制約
- 全色で黒文字テキストのWCAGコントラスト比7.0以上を確保
- 1列8色のグリッド表示（9行×8列 = 72色ぴったり）
- パレット上部に選択中の色名を表示

### 定義場所
- 色の値: `TabbedMemoListView.swift` の `tabColors` 配列
- 色の名前: 同ファイルの `tabColorNames` 配列
- 色相計算用RGB: `MainView.swift` の `hueFromColorIndex()` 内テーブル（tabColorsと同じ値を維持すること）
